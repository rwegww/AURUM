# Quy tắc làm việc cho Codex

## Commit message

- Khi tạo commit hoặc đề xuất commit message, luôn viết bằng tiếng Việt.
- Không dùng tiếng Anh trong phần mô tả commit.
- Dùng format:

  <type>: <mô tả ngắn bằng tiếng Việt>

- Các type được dùng:
  - feat: thêm chức năng mới
  - fix: sửa lỗi
  - refactor: tối ưu hoặc sắp xếp lại code
  - style: chỉnh giao diện hoặc định dạng
  - docs: cập nhật tài liệu
  - chore: thay đổi nhỏ, cấu hình, dọn dẹp
  - test: thêm hoặc sửa kiểm thử

## Ví dụ commit hợp lệ

- feat: thêm chức năng tạo câu hỏi bằng AI
- fix: sửa lỗi lưu trạng thái bài học
- refactor: tách logic xử lý quiz trong trang admin
- style: chỉnh màu giao diện phù hợp chế độ sáng tối
- docs: cập nhật hướng dẫn sử dụng hệ thống
